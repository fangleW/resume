# 构建单页滚动式 React 个人简历 (Vite 方案)

根据你的需求调整，我们将采用**单页滚动 (One Page Scroll)** 的形式，让用户在一个页面内浏览所有内容，体验更加流畅。

## 1. 框架选型建议：为什么选择 Vite？
经过分析，我建议**继续使用 Vite**，而不使用 Modern.js 或 Umi。
-   **轻量级**: 简历项目通常是纯静态的，Vite 的打包产物更小，加载速度更快。
-   **无冗余**: Modern.js 和 Umi 是优秀的企业级框架，内置了大量路由管理、SSR、微前端等功能，对于单页简历来说属于“过度工程化”，会增加不必要的复杂度和学习成本。
-   **灵活性**: Vite + React 的组合非常纯粹，方便我们自由集成 Framer Motion 和 Tailwind。

## 2. 项目初始化
-   **核心**: Vite + React + TypeScript。
-   **配置**: 路径别名 `@` 配置，ESLint/Prettier 代码规范配置。

## 3. 依赖调整
-   **移除**: `react-router-dom` (不再需要复杂路由)。
-   **新增**: `react-scroll` (用于处理导航栏点击后的平滑滚动跳转)。
-   **保留**:
    -   `tailwindcss` + `postcss` + `autoprefixer` (样式)。
    -   `framer-motion` (滚动入场动画)。
    -   `lucide-react` (图标)。
    -   `clsx` + `tailwind-merge` (样式组合)。

## 4. 目录与组件结构
我们将采用 **Section-based** 的结构，每个版块作为一个独立的组件：
```text
src/
├── components/     # 通用 UI 组件 (Button, SectionTitle)
├── layouts/        # 布局组件 (Navbar, Footer)
├── sections/       # 页面主要版块
│   ├── Hero.tsx    # 首屏
│   ├── About.tsx   # 关于
│   ├── Skills.tsx  # 技能
│   ├── Projects.tsx# 项目
│   └── Contact.tsx # 联系
└── App.tsx         # 组合所有 Section
```

## 5. 开发计划
### Phase 1: 骨架搭建
-   初始化项目与 Tailwind CSS。
-   实现 `Navbar`：固定在顶部，包含指向各 Section 的锚点链接（如 `#about`, `#projects`），支持点击平滑滚动。

### Phase 2: 版块开发
-   **Hero Section**: 全屏高度，包含醒目的个人介绍和动态背景。
-   **Content Sections**: 依次开发 About, Skills, Projects, Contact。
    -   *设计点*: 使用 Tailwind 的 Grid/Flex 布局实现响应式排版。

### Phase 3: 动画增强 (Framer Motion)
-   **Scroll Reveal**: 为每个 Section 添加 `whileInView` 属性，实现“滚动到视口时淡入/上浮”的效果。
-   **Stagger Effect**: 技能列表和项目卡片使用交错动画，依次出现。

## 6. 质量保证
-   TypeScript 类型检查。
-   响应式适配测试（Mobile/Tablet/Desktop）。
